Emmagina Backend v2.1 - CMS Editor del Sitio estable

Cambio principal:
- El Editor del Sitio deja de depender de la colección antigua pages.
- Usa una colección limpia llamada site_pages para evitar errores heredados, duplicados o índices únicos dañados.
- Las rutas públicas y admin se mantienen iguales:
  /api/pages/home
  /api/admin/pages
  /api/admin/pages/_repair
  /api/admin/pages/_diagnostic

Orden:
1. Reemplazar backend sin borrar .git.
2. npm install
3. npm run check
4. git add .
5. git commit -m "Emmagina backend v2.1 CMS stable"
6. git push
7. Probar /api/pages/home después del deploy.

Nota:
La colección antigua pages puede quedar en MongoDB sin tocarse. Esta versión usa site_pages.
