EMMAGINA BACKEND v1.1 estable

Tipo de servicio recomendado en Render:
- Web Service
- Build Command: npm install
- Start Command: npm start

Variables mínimas de producción:
NODE_ENV=production
MONGODB_URI=...
JWT_SECRET=...
SECURITY_LOG_SALT=...
TRUST_PROXY_HOPS=1
FRONTEND_URLS=https://emmagina-frontend.onrender.com
PUBLIC_FRONTEND_URL=https://emmagina-frontend.onrender.com
PUBLIC_BACKEND_URL=https://emmagina-backend.onrender.com
CLOUDINARY_CLOUD_NAME=...
CLOUDINARY_API_KEY=...
CLOUDINARY_API_SECRET=...
CLOUDINARY_PRODUCT_FOLDER=emmagina/productos
CLOUDINARY_CUSTOMIZATION_FOLDER=emmagina/personalizaciones

Rutas clave:
GET /api/health
GET /api/uploads/estado
GET /api/productos
GET /api/producto        (alias temporal)
GET /api/catalogo        (alias temporal)
POST /api/uploads/productos  (subida de imágenes desde admin, requiere admin/gestor)

Comandos útiles:
npm run check
npm run create-admin
npm run seed-emmagina

Para crear usuario admin:
1. Configura ADMIN_EMAIL y ADMIN_PASSWORD en Render temporalmente.
2. Ejecuta en Render Shell: npm run create-admin
3. Elimina ADMIN_PASSWORD después de crear el usuario.

Para cargar productos base:
- Ejecuta en Render Shell: npm run seed-emmagina
